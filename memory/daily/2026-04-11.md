## 세카이 라우터 분리 작업
**문제:** 세카이 채널에서 main 봇이 의사결정 텍스트를 직접 출력하는 버그 재발.
**원인:** sonnet 모델이 SOUL.md NO_REPLY 지시를 안정적으로 따르지 못함.

**해결책: sekai-router 에이전트 분리**
- `sekai-router` 에이전트 신설 (`workspace-sekai`, sonnet)
- bindings: `discord peer=channel:1485510333115273339` → `sekai-router`
- main은 헤드쿼터 등 나머지 채널만 담당
- `daily-weather` 크론: agentId → `sekai-router`로 변경
- main 하트비트 비활성화 (every=0m), sekai-router가 하트비트 담당
- SOUL.md 강화: "텍스트 출력 = 시스템 오류" 프레이밍

**결과:** 1회 테스트 통과 (대리 발화 + NO_REPLY 정상).
**잔여 리스크:** sonnet 특성상 컨텍스트 누적 시 재발 가능. 재발 시 sekai-router를 opus로 전환 예정.
